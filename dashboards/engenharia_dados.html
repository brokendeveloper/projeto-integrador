🔍 Deduplicação

SQL

SQLite — Persistência Relacional Secundária

O ETL grava simultaneamente no MongoDB Atlas (primário) e no SQLite local (análise offline).

CREATE TABLE IF NOT EXISTS contratos ( id INTEGER PRIMARY KEY, numero_controle_pncp TEXT UNIQUE, objeto_contrato TEXT, valor_inicial REAL, data_publicacao_pncp TEXT, data_vigencia_inicio TEXT, data_vigencia_fim TEXT, orgao_cnpj TEXT, orgao_razao_social TEXT, etl_timestamp TEXT, source TEXT ); -- Deduplicação: INSERT OR IGNORE INSERT OR IGNORE INTO contratos (...) VALUES (?, ?, ?, ...); -- Exemplo de análise ad-hoc SELECT orgao_razao_social, COUNT(*) total, AVG(valor_inicial) media FROM contratos WHERE valor_inicial <= 80000 GROUP BY orgao_razao_social ORDER BY total DESC;
MongoDB vs SQLite
SchemaDinâmico (JSON) vs Fixo (relacional)
orgaoEntidadeAninhado vs Flat (orgao_cnpj / razao_social)
DeduplicaçãoUpsert vs INSERT OR IGNORE
AcessoMotor async vs sqlite3 sync
UsoAPI + PySpark vs Offline + SQL ad-hoc

💡 Por que SQLite?

Permite análise exploratória dos dados sem depender de conexão com o MongoDB Atlas. Útil para demos offline e queries SQL rápidas durante o desenvolvimento.

Streaming vs Batch

Por que Batch? Roadmap Kafka

✅

Batch — escolha correta para o PNCP

A API do PNCP é pull-based de atualização periódica. Órgãos publicam contratos em lotes diários ou semanais — não há eventos em tempo real. Kafka sobre essa fonte seria over-engineering sem ganho operacional.

⚡

Near-realtime já existe

Os endpoints GET /analytics/estatisticas, top-orgaos e contratos-mei consultam o MongoDB diretamente via Aggregation Pipeline — dados atualizados a cada request sem aguardar PySpark.

MCP Server

Model Context Protocol — 4 Ferramentas

O MCP expõe as ferramentas de analytics em dois transports: HTTP REST e protocolo MCP nativo (Claude Desktop).

buscar_contratos
Busca por palavra-chave no objeto do contrato usando regex case-insensitive no MongoDB.
palavra_chave* limite=10
estatisticas_contratos
Retorna total de contratos, valor médio, máximo e mínimo via MongoDB $group aggregation.
sem parâmetros
top_orgaos
Órgãos com mais contratos publicados. groupBy + count + sort + limit no MongoDB.
n=5
contratos_favoraveis_mei
Contratos com valorInicial ≤ R$ 80.000. Preferência MEI por LC 123/2006 Art. 48.
limite=10

HTTP REST

GET /mcp/ferramentas — lista as 4 tools com input_schema
POST /mcp/invocar — executa por nome

Resiliência

Erros Mapeados — Pipeline não para

Cada camada tem isolamento de erro. Uma falha em um módulo não derruba a API inteira.

Ponto de falha Tratamento Impacto
API PNCP timeout Retry exponencial 3× (tenacity, backoff 2–10s) → retorna parcial ETL parcial
MongoDB offline (ETL) ConnectionFailure capturada, pipeline encerrado com ERROR ETL falha; API ok
MongoDB offline (API) Motor retorna erro; FastAPI retorna 500 com mensagem humanizada Endpoint falha; demais ok
PySpark sem dados no Bronze if pdf.empty: spark.stop(); return — sem write Gold não atualizado
CSVs Gold ausentes { "disponivel": false } — sem erro 500 Dashboard sem Spark
Tool do chatbot falha Exception capturada; string de erro enviada ao Claude; Claude informa usuário Resposta degradada
ANTHROPIC_API_KEY ausente Retorna ChatResponse com mensagem de configuração antes de chamar a API Léo indisponível; API ok
Bulk write parcial (MongoDB) BulkWriteError logado com exc.details; próxima run re-tenta via upsert Auto-recuperável
